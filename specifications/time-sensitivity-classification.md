# Time-Sensitivity Classification

## Purpose

This specification defines time-horizon labels for lessons, patterns, planning signals, and principles.

Business lessons do not all age the same way. Some are short-lived tactics. Others are structural observations.

## Labels

Use these labels when useful:

| Label | Meaning |
|---|---|
| `short-term` | Depends on temporary timing, market condition, crisis, fad, arbitrage, or tactical opening. |
| `medium-term` | Useful across a limited business cycle, technology cycle, leadership period, or industry phase. |
| `long-term` | Appears durable across multiple cycles but still depends on context. |
| `structural` | Tied to recurring structural features such as incentives, capital constraints, information asymmetry, distribution, governance, or human behavior. |
| `unknown` | Evidence is insufficient to classify. |

## Application Rule

Apply time-sensitivity labels to:

- lessons when transferability is discussed;
- patterns before acceptance;
- principles before promotion;
- SBA planning signals when timing materially affects interpretation.

## Obsolescence Rule

If a lesson depends on obsolete law, technology, financing, labor conditions, or ethics, mark the transferability limits clearly.

Use existing transferability labels where appropriate:

- `still-valid`
- `context-dependent`
- `obsolete`
- `illegal-or-unethical-today`
- `unknown`

## Confidence Rule

Time-horizon classification should not raise confidence. It only describes expected durability.
