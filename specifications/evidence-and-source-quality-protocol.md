# Evidence and Source Quality Protocol

## Purpose

This protocol defines how the repository labels evidence quality before facts, claims, lessons, patterns, or principles are promoted.

Sources provide evidence. They are not automatically true.

## Evidence Labels

Use these labels when source support needs to be classified:

| Label | Meaning |
|---|---|
| `primary-supported` | Supported by primary records or direct contemporaneous documentation. |
| `participant-perspective` | Supported by a founder, executive, investor, employee, or other direct participant's account. Useful, but not neutral proof. |
| `reported` | Supported by reputable journalism or external reporting. |
| `scholarly-analysis` | Supported by academic, business-school, peer-reviewed, or university-press analysis. |
| `secondary-synthesis` | Supported by a well-sourced book, biography, or historical synthesis. |
| `orientation-only` | Useful for leads or orientation, not for promotion. |
| `disputed` | Material sources conflict or interpretation is contested. |
| `model-based` | Derived from analytical modeling, financial interpretation, or reconstruction rather than direct historical statement. |
| `uncertain` | Evidence is incomplete, weak, ambiguous, or not yet checked. |

## Promotion Rule

Important facts and claims should preserve source quality labels.

A repeated weak source does not become strong by repetition.

## Cross-Check Rule

Important claims should be checked against more than one source type when possible.

If cross-checking is not yet possible, mark the claim as candidate, developing, single-source, disputed, or uncertain as appropriate.

## Source Laundering Rule

Do not cite a secondary source as if it were primary evidence when the secondary source merely repeats another account.

Trace claims back to the strongest available source tier.
