# Phase Plan v2 (Merged)

This document supersedes the original Phase Plan and incorporates all approved architectural amendments.

## Major Changes From Original Phase Plan

### New Governance Concepts

- Negative Knowledge System
- Assumptions Registry
- Competing Interpretations
- Pattern Failure Conditions
- Time-Sensitivity Classification

### New Early Phases

#### Phase 0.5 — Minimal Validation Foundation

Create a lightweight validator before repository expansion.

Checks:
- INDEX presence
- path validity
- CSV schema validation
- ID uniqueness

#### Phase 1.5 — LLM Governance Gate

Implement LLM governance before large-scale repository growth.

Coverage:
- prompt injection
- hallucinated sources
- unsupported claims
- excessive agency
- write-action approval
- source laundering

### Strengthened Knowledge Model

Knowledge ladder becomes:

Sources -> Facts -> Claims -> Lessons -> Patterns -> Principles

Supported by:

- assumptions
- competing interpretations
- counterexamples
- failure conditions
- rejected knowledge records

### Negative Knowledge System

Track:

- rejected facts
- rejected claims
- rejected lessons
- retired patterns
- rejected principles

Purpose:
Preserve what failed, why it failed, and what evidence disproved it.

### Pattern Requirements

Every accepted pattern must include:

- supporting evidence
- counterexamples
- transferability
- assumptions
- failure conditions
- competing interpretations

### Principle Requirements

No principle may be accepted unless:

- strong supporting patterns exist
- counterexamples are documented
- assumptions are documented
- failure conditions are documented
- competing interpretations are considered
- provenance remains traceable
